What this app is

Chore Manager lets households manage chores, points, streaks, rewards, and roles (admin/member/child). Households are invite-based; admins manage memberships and approvals.

Accounts and roles

  • Account creation requires a username; admins/parents also need an email. Child accounts can be created without email.
  • Joining a household requires a valid invite code. Admins can invite/remove members and adjust roles.
  • Users must keep credentials secure and use the app lawfully.

User content and responsibilities

  • Households own and control their content: chores, rewards, photos, and other entries.
  • Only upload content you have rights to; no unlawful, abusive, or infringing material.
  • Photo uploads for verification are stored; admins can manage/delete via app flows (if available) or by request.

Privacy overview

We only collect the info you enter (account fields) and in-app activity (chores, points, rewards, notifications, optional photos). Child accounts can be created without email; no additional data is collected for children. See the Privacy page for details.

Availability and changes

  • The service/software is provided “as is,” with no uptime guarantees.
  • Features may change; check release notes for updates.

Security and API

  • Passwords are hashed using Django defaults. Household access is invite-based.
  • API authentication is not implemented in the current code—limit deployments accordingly until auth is added.

Liability and warranty

Provided “as is” without warranties; liability is limited to the maximum extent allowed by law.

Termination and data handling

If an account or household is removed, associated data may be deleted. Define and document your deletion/export process for production use.
